# 개념

## 벡터화란?

벡터화는 이미지, 텍스트, 음성 같은 비정형 데이터를 숫자 배열 형태로 변환하는 과정입니다.

검색 시스템 관점에서 벡터화는 단순한 형식 변환이 아니라 데이터를 비교 가능한 표현으로 바꾸는 작업입니다. 사람이 보기에는 서로 다른 이미지와 문장도 모델이 같은 방식으로 비교할 수 있도록 만드는 전처리 단계입니다.

## 벡터화 방식의 분류

| 방식 | 설명 | 특징 |
| --- | --- | --- |
| 임베딩(Embedding) | 데이터의 의미, 관계, 맥락을 반영해 저차원 연속 벡터 공간에 표현하는 방식 | 의미 기반 검색, 추천, 유사도 비교에 적합 |
| 원-핫 인코딩(One-Hot Encoding) | 범주형 데이터를 0과 1로만 구성된 벡터로 표현하는 방식 | 단순하고 해석이 쉽지만 의미적 유사성은 반영하지 못함 |
| 카운트 벡터(Count Vector) | 문서나 문장에서 단어 등장 횟수를 숫자로 표현하는 방식 | 구현이 쉽지만 단어 순서와 문맥은 반영하지 못함 |
| 티에프 아이디에프(TF-IDF) | 문서 내 빈도와 전체 문서에서의 희소성을 함께 반영하는 방식 | 검색과 문서 분류에서 오래 쓰인 표준 방식 |
| 워드투벡(Word2Vec) | 단어를 의미적으로 비슷한 단어끼리 가까운 벡터가 되도록 학습하는 방식 | 단어 수준 의미 표현에 강함 |
| 패스트텍스트(FastText) | Word2Vec을 확장해 문자 단위 조각까지 고려하는 방식 | 희귀 단어, 오타, 형태 변화가 있는 언어에 상대적으로 유리함 |
| 독투벡(Doc2Vec) | 문장이나 문서 전체를 하나의 벡터로 표현하는 방식 | 문서 단위 비교에 적합하지만 최근에는 Transformer 계열에 많이 대체됨 |
| 버트 임베딩(BERT Embedding) | Transformer 기반으로 문맥까지 반영해 벡터를 생성하는 방식 | 문맥 이해 능력이 강하지만 계산 비용이 비교적 큼 |
| 센텐스 트랜스포머즈(Sentence Transformers) | 문장 전체를 유사도 비교에 적합한 벡터로 만들도록 튜닝된 모델 계열 | 문장 검색과 의미 검색에 강함 |
| 이미지 임베딩(Image Embedding) | 이미지의 형태, 색상, 패턴, 질감 등을 반영한 벡터 표현 | 이미지 검색, 분류, 추천에 사용됨 |
| 멀티모달 임베딩(Multimodal Embedding) | 이미지와 텍스트처럼 서로 다른 데이터를 같은 공간에서 비교 가능하게 만드는 방식 | 이미지-텍스트 매칭, 커머스 검색에 적합 |

## 멀티모달 임베딩이 필요한 이유

일반적인 단일 모달 임베딩 모델은 하나의 데이터 타입을 벡터화하는 데 최적화되어 있습니다.

* 텍스트 → 텍스트 임베딩
* 이미지 → 이미지 임베딩

이 방식은 동일한 타입의 데이터 간 유사도 비교에는 효과적입니다.

* 텍스트 임베딩 → 문장, 키워드, 상품명 간 유사도 비교
* 이미지 임베딩 → 이미지 간 시각적 유사도 비교

하지만 단일 모달 임베딩만으로는 이미지와 텍스트를 동일한 기준에서 비교하기 어렵습니다.
예를 들어 사용자가 상품 이미지를 업로드했을 경우, 해당 이미지와 상품명, 브랜드, 카테고리와 같은 텍스트 정보를 직접적으로 연결해 비교하는 데 한계가 있습니다.

이미지 검색 서비스에서는 단순히 시각적으로 유사한 상품을 찾는 것만으로는 충분하지 않습니다.
색상이나 형태가 유사하더라도 실제 카테고리가 다를 수 있으며, 브랜드명이나 상품명에 포함된 의미 정보 또한 검색 결과의 품질에 중요한 영향을 미칩니다.

멀티모달 임베딩은 이미지와 텍스트를 동일한 벡터 공간에 표현함으로써, 다음과 같은 검색 흐름을 가능하게 합니다.

* 입력 이미지와 상품 이미지 간 유사도 비교
* 입력 이미지와 상품명, 브랜드, 카테고리 텍스트 간 의미 기반 비교
* 텍스트 검색어와 상품 이미지 간 유사도 비교
* 이미지 유사도와 텍스트 의미 정보를 결합한 후보 검색

한편, 가격, 판매 상태, 품절 여부, 노출 가능 여부와 같은 값은 임베딩 벡터 자체보다는 필터링 및 랭킹 재계산 단계에서 활용하는 것이 적합합니다.

결론적으로 이 서비스는 멀티모달 임베딩을 통해 이미지와 텍스트를 하나의 검색 흐름으로 통합하고, 메타데이터 기반 필터링 및 스코어링을 함께 적용하여 최종 검색 정확도를 높이는 구조가 바람직합니다.

> <b>모달과 모델의 차이점</b>  
> 모달 ≠ 모델  
> - 모달: 텍스트, 이미지 같은 데이터 형태  
> - 모델: 그 데이터를 처리하는 AI 시스템  

## CLIP

CLIP(Contrastive Language-Image Pretraining)은 이미지와 문장을 같은 좌표계에 놓고 학습한 신경망 모델입니다.

- 이미지와 텍스트를 같은 임베딩 공간에 매핑합니다.
- 이미지가 주어졌을 때 관련성이 높은 텍스트를 찾거나, 텍스트가 주어졌을 때 관련 이미지 검색에 활용할 수 있습니다.

|     |     |     |     |     |
| --- | --- | --- | --- | --- |
| 모델명 | HuggingFace Full Name | 이미지 인코더 | 입력 해상도 | 특징  |
| ViT-B/32 | `openai/clip-vit-base-patch32` | ViT (Base) | 224 | 표준 모델 |
| ViT-B/16 | `openai/clip-vit-base-patch16` | ViT (Base) | 224 | 더 정밀 |
| ViT-L/14 | `openai/clip-vit-large-patch14` | ViT (Large) | 224 | 고성능 |
| ViT-L/14@336 | `openai/clip-vit-large-patch14-336` | ViT (Large) | 336 | 고해상도 |

- 참고 자료
  - https://arxiv.org/abs/2103.00020
  - https://github.com/openai/CLIP

## OpenCLIP

- OpenAI CLIP의 오픈소스 재구현 모델.
- LAION 대규모 이미지-텍스트 데이터셋으로 재학습된 모델을 사용할 수 있습니다.
- 이미지 검색과 텍스트-이미지 매칭에 적합합니다.

|     |     |     |     |     |
| --- | --- | --- | --- | --- |
| 모델명 | HuggingFace Full Name | 이미지 인코더 | 입력 해상도 | 특징  |
| ViT-B/32 (LAION) | `laion/CLIP-ViT-B-32-laion2B-s34B-b79K` | ViT (Base) | 224 | 대규모 데이터 학습 |
| ViT-H/14 | `laion/CLIP-ViT-H-14-laion2B-s32B-b79K` | ViT (Huge) | 224 | 매우 높은 성능 |

> **LAION 이란?**  
> LAION(Large-scale Artificial Intelligence Open Network)은 독일 기반 비영리 AI 단체로, 대규모 이미지와 텍스트 데이터셋을 공개합니다.
> 
> - 데이터 구조: 이미지 URL + 텍스트 설명(Caption)
> - 주요 규모:
> - LAION-400M: 약 4억 개
> - LAION-2B: 약 20억 개
> - LAION-5B: 약 58억 개

## FashionCLIP

- OpenAI CLIP 기반 일반 이미지-텍스트 매칭이 아닌 패션 상품 검색/분류에 특화 모델.
- 패션 상품 이미지 + 상품명/설명 텍스트 를 학습함.
- 일반 CLIP 대비:
  - 의류 카테고리 구분 (셔츠, 원피스, 아우터 등)
  - 스타일 표현 (casual, formal, street 등)
  - 색상/핏/디테일 인식
- 패션 상품 검색 정확도 개선

|     |     |     |     |     |
| --- | --- | --- | --- | --- |
| 모델명 | HuggingFace Full Name | 이미지 인코더 | 입력 해상도 | 특징  |
| FashionCLIP | `patrickjohncyh/fashion-clip` | ViT-B/32 기반 | 224 | 패션 도메인 특화 |
